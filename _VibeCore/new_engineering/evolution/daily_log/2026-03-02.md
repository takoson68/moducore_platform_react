# Daily Evolution Log

Date: 2026-03-02
Scope: moducore_platform/platform/

## Why / What / Impact（Human-readable）
- Why: 發生一次 project-specific API 對齊需求錯誤回灌到共享層的實作偏移，需要正式留痕。
- What: 先撤回 `platform/frontend/src/app/api/*` 的 shared-layer 變更，並新增一份 candidate convention 約束此類修改邊界。
- Impact: 之後若再次處理前後端 API 對齊，應先在 module / project 自有 API adapter / service 處理，避免再把單一專案或單一模組需求推進平台共享層。
- Why: `flowCenter` 登入流程偏離平台既有模式，額外建立 project-local auth truth，導致登入成功後畫面又回未登入。
- What: 將 `flowCenter` auth flow 收斂回平台 `authApi + authStore` 的單一真相模式，移除 project-local auth truth。
- Impact: 之後建立新專案時，auth 架構應直接沿用平台標準 auth pattern，避免再次因第二份 session state 與 project-specific auth 判定鏈造成覆蓋、401 自動登出與 UI 閃回。
- Why: 需要把 `project-b/modules/` 的實際運作鏈抽象成平台規則，避免之後新增專案時又誤解 `modules/index.js`、`boot.js` 與 router/register 的責任分工。
- What: 將「模組收集 -> discover -> visibility -> install -> register -> router/buildRoutes -> runtime ready」整理成 candidate convention。
- Impact: 之後討論新專案 module 架構時，可以直接依這條 pipeline 判斷哪些責任應落在 `modules/index.js`、哪些應留在 `boot/register/router`。
