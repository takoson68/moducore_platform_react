# 餐廳地圖產生器執行步驟

## 文件定位
本文件用於將《餐廳地圖產生器》設計文件轉換為可直接執行的開發步驟。

目標不是重新定義規格，而是依既有規格拆出實作順序，讓餐廳地圖產生器新模組可分階段完成。

## 1. 執行原則

- 本模組為新模組開發，不直接修改既有桌位管理模組。
- 開發期間地圖資料與桌位資料需維持分流。
- 既有桌位管理模組僅保留未來資料庫對接需求。
- 功能驗證完成後，再以替換與淘汰方式移除舊桌位管理模組。
- 所有座標一律使用世界座標。
- 畫面實作採用 `SVG + HTML` 結構，不使用 HTML `<canvas>` 編輯器模型。

## 2. 開發階段

## 2.0 里程碑進度

本模組可依下列里程碑追蹤整體專案進度。

- `Milestone 1`：完成模組骨架與地圖建立流程
- `Milestone 2`：完成 `Map Edit Mode` 與基礎繪圖工具
- `Milestone 3`：完成草稿 / 正式儲存、重新載入與多地圖切換
- `Milestone 4`：完成 `Table Edit Mode` 與桌位互動
- `Milestone 5`：完成資料交換格式、櫃台模組匯入與資料庫對接

建議進度標示方式：

- `未開始`
- `進行中`
- `已完成`
- `阻塞`

建議里程碑對照：

- `Milestone 1` 對應 `2.1`、`2.2`
- `Milestone 2` 對應 `2.3`、`2.4`、`2.5`
- `Milestone 3` 對應 `2.6`、`2.7`
- `Milestone 4` 對應 `2.8`
- `Milestone 5` 對應 `2.9`

### 2.1 第一階段：建立模組骨架

里程碑：`Milestone 1`
建議狀態欄位：`未開始 / 進行中 / 已完成 / 阻塞`

1. 建立餐廳地圖產生器新模組入口。
2. 建立頁面層 `Workspace` 基本結構。
3. 建立模式切換區與上方工具列區塊。
4. 建立 `SVG Root`、`Map Layer`、`Table Layer`、`UI Layer / Overlay Layer` 基本容器。
5. 建立地圖清單或等效切換入口，支援 `active map` 概念。

完成條件：

- 可以進入餐廳地圖產生器頁面。
- 頁面上可看到 `Workspace` 與基礎 UI 結構。
- 尚未實作繪圖功能前，結構與模式切換位置已固定。

### 2.2 第二階段：建立地圖流程

里程碑：`Milestone 1`
建議狀態欄位：`未開始 / 進行中 / 已完成 / 阻塞`

1. 建立「新增地圖」按鈕。
2. 建立新增地圖表單。
3. 驗證地圖名稱、寬度、高度必填且為有效數值。
4. 建立地圖後，自動指定為 `active map`。
5. 建立空白地圖顯示，並加入 `grid` 視覺參考。
6. 建立新地圖後，預設進入 `Edit Mode` 下的 `Map Edit Mode`。

完成條件：

- 可以建立新地圖。
- 新地圖建立後可正確顯示於 `Workspace`。
- 第一版同一時間只有一張 `active map` 進入編輯。

### 2.3 第三階段：Map Edit Mode 基礎互動

里程碑：`Milestone 2`
建議狀態欄位：`未開始 / 進行中 / 已完成 / 阻塞`

1. 建立 `Map Edit Mode` 狀態。
2. 建立地圖工具列：`Polyline`、`Rectangle`、`Circle`、`Text`。
3. 進入 `Map Edit Mode` 後，預設不自動啟用任何工具。
4. 點擊工具後，才開始建立對應物件。
5. 建立 `Object Edit Mode` 狀態與工具鎖定邏輯。
6. 建立 `active object` 狀態。

完成條件：

- `Map Edit Mode` 可正確切換。
- 工具啟用、工具鎖定、空白區點擊規則可正常運作。

### 2.4 第四階段：Polyline 工具

里程碑：`Milestone 2`
建議狀態欄位：`未開始 / 進行中 / 已完成 / 阻塞`

1. 建立 `Polyline` 繪製流程。
2. 建立世界座標下的節點記錄。
3. 實作水平吸附、垂直吸附、起點吸附。
4. 建立 `Polyline` 完成條件：至少兩個節點。
5. 建立 `Polyline` 的 `Object Edit Mode`。
6. 支援節點拖曳、節點刪除、插入節點。
7. 支援 segment `hover` 顯示單一空心控制點。
8. 支援將 segment 由 `line` 轉為 `quadratic`。
9. 支援拖曳曲線控制點調整曲率。
10. 支援將曲線段還原為直線段。
11. 支援刪除整條折線。
12. 建立繪製中暫存狀態 `pendingPolyline`。

完成條件：

- 可建立折線。
- 可編輯折線節點。
- 可將單一線段轉為簡單曲線並再次調整。
- 可刪除整條折線。
- 未完成折線不會直接寫入正式 `objects`。

### 2.5 第五階段：Rect / Circle / Text 工具

里程碑：`Milestone 2`
建議狀態欄位：`未開始 / 進行中 / 已完成 / 阻塞`

1. 建立 `rect` 繪製流程。
2. 建立 `circle` 繪製流程，資料以 `x / y / width / height` 表示。
3. 建立有效拖曳距離判斷，避免無效物件建立。
4. 建立 `text` 建立流程與文字輸入入口。
5. 禁止建立空白文字物件。
6. 建立 `rect / circle / text` 的 `Object Edit Mode`。
7. 支援移動、調整大小、刪除。

完成條件：

- `rect / circle / text` 都可建立。
- `rect / circle / text` 可編輯與刪除。
- `text resize` 只影響文字框，不影響字級。

### 2.6 第六階段：地圖儲存與草稿

里程碑：`Milestone 3`
建議狀態欄位：`未開始 / 進行中 / 已完成 / 阻塞`

1. 建立正式資料與編輯中暫存狀態分流。
2. 建立草稿存檔格式。
3. 建立正式儲存格式。
4. 儲存時寫入世界座標資料。
5. 重新載入草稿時恢復 `draftState`。
6. 儲存成功後清除 `active map` 的未儲存狀態。
7. 建立未儲存變更提示流程。

完成條件：

- 可草稿存檔。
- 可正式儲存。
- 可重新載入並恢復編輯。

### 2.7 第七階段：多地圖切換

里程碑：`Milestone 3`
建議狀態欄位：`未開始 / 進行中 / 已完成 / 阻塞`

1. 建立 `active map` 切換流程。
2. 建立未儲存切換提示。
3. 建立刪除 `active map` 流程。
4. 刪除後清理 `active object`、`draftState`、`working mode`。
5. 若仍有其他地圖，自動切到下一張可用地圖。
6. 若已無任何地圖，回到空狀態。

完成條件：

- 多地圖切換穩定。
- 刪除地圖時狀態清理完整。

### 2.8 第八階段：Table Edit Mode

里程碑：`Milestone 4`
建議狀態欄位：`未開始 / 進行中 / 已完成 / 阻塞`

1. 建立 `Table Edit Mode` 狀態。
2. 建立 `Table Layer`。
3. 建立 `tables` 集合與 `active table` 狀態。
4. 建立 `Table` 工具。
5. 支援新增桌位、選取桌位、移動桌位、調整大小、刪除桌位。
6. 建立 `label` 預設命名流程，如 `T1`、`T2`、`T3`。
7. 切換 `Map Edit Mode` / `Table Edit Mode` 時，清除前一模式 active 狀態。

完成條件：

- `Table Edit Mode` 能獨立操作桌位。
- 不會跨 layer 誤操作 `objects`。

### 2.9 第九階段：櫃台模組匯入與資料對接

里程碑：`Milestone 5`
建議狀態欄位：`未開始 / 進行中 / 已完成 / 阻塞`

1. 建立地圖資料對外交換格式。
2. 建立草稿與正式儲存格式轉換層。
3. 建立櫃台模組可匯入的資料格式。
4. 建立與既有資料庫的映射層。
5. 確保地圖編輯資料模型不被營運欄位直接污染。

完成條件：

- 櫃台模組可匯入已完成地圖資料。
- 新模組與既有資料庫之間以轉換層對接。

## 3. 驗收順序

### 3.1 第一批驗收

- 新增地圖
- 空白地圖顯示
- `Map Edit Mode`
- `Polyline`
- `rect / circle / text`
- 物件編輯與刪除

### 3.2 第二批驗收

- 草稿存檔
- 正式儲存
- 重新載入
- 多地圖切換
- 未儲存提示

### 3.3 第三批驗收

- `Table Edit Mode`
- 桌位新增 / 編輯 / 刪除
- 資料交換格式
- 櫃台模組匯入

## 4. 禁止事項

- 不得直接在既有桌位管理模組內擴寫餐廳地圖產生器功能。
- 不得將 `tables` 混入 `objects` 作為同一集合處理。
- 不得以畫面座標作為正式資料座標。
- 不得讓 `Map Edit Mode` 與 `Table Edit Mode` 共用同一套命中邏輯而不分流。
- 不得在第一版提前綁定完整營運欄位，污染地圖編輯資料模型。

## 5. 執行結果

當以上步驟完成後，餐廳地圖產生器應具備：

- 可建立與切換多張地圖
- 可在 `Map Edit Mode` 下完成地圖繪製
- 可草稿存檔與正式儲存
- 可重新載入並繼續編輯
- 可在第二階段接入 `Table Edit Mode`
- 可作為後續櫃台功能模組的地圖基礎
