# 點餐系統顧客體驗改進清單

更新日期：2026-03-04

## 1. 文件目的

本文件從「顧客實際使用點餐系統」的角度出發，
整理目前 DineCore 在體驗層仍值得優先改善的問題，
並給出建議優先順序，方便後續轉成產品任務或 UI / UX 實作項目。

---

## 2. 主要問題與改進方向

### A. 流程感不夠明確

目前顧客雖然能完成點餐，
但不一定能直覺理解自己目前處於哪一個階段：

- 正在選餐
- 正在確認本批
- 已送出
- 等待製作 / 取餐

在多批次送單模型下，這個問題更明顯。

建議改進：

- 在 `menu / cart / checkout / success / tracker` 補上更明確的流程提示
- 顯示「目前是第幾批」
- 顯示「前一批是否已送出」
- 讓顧客知道目前編輯的是「本批」而不是整張桌的全部內容

---

### B. `合併訂單` 文案不夠直覺

`合併訂單` 這個詞偏系統語言，
一般顧客第一次使用時不容易立刻理解。

建議改進：

- 改成更直覺的顧客語言，例如：
  - `確認本批餐點`
  - `前往確認`
  - `檢查本批訂單`

文案原則：

- 不要讓顧客先理解系統模型，才會知道按鈕要做什麼
- 優先使用「下一步」與「我要送出這批餐點」的語感

---

### C. 送單成功後的安心感不足

顧客送單後最在意的是：

- 店家有沒有收到
- 這一批送了什麼
- 大概要等多久

若畫面只顯示訂單編號與狀態，
安心感仍然不足。

建議改進：

- 成功頁顯示：
  - 第幾批已送出
  - 本批品項數
  - 本批金額
  - 店家已收到
  - 預估等待時間
- 追單頁持續顯示這些關鍵資訊

---

### D. 多人共桌體驗偏技術導向

目前像 `1號顧客 / 2號顧客` 這種標記，
雖然對系統歸屬有幫助，
但對顧客來說較不自然。

建議改進：

- 前台弱化技術編號感
- 優先顯示 `你的餐點`
- 若需要區分多人，可提供快速暱稱，例如：
  - `我`
  - `朋友 A`
  - `小孩`

---

### E. 異常狀態提示還不夠顧客導向

例如：

- 桌號停用
- 暫停接單
- 點餐 session 失效
- 網路短暫失敗

這些情況若只回系統代碼或過度技術化訊息，
顧客通常只會感覺系統故障。

建議改進：

- 所有錯誤文案保持短句
- 每個提示都盡量附帶下一步
- 原則如下：
  - 告訴顧客發生什麼
  - 告訴顧客接下來怎麼做

示例方向：

- `此桌目前暫停接單，請稍後再試`
- `點餐狀態已失效，請重新掃描桌邊 QR Code`
- `網路暫時不穩，請稍後重新送出`

---

### F. 菜單頁還不夠像在幫顧客做決定

目前菜單功能完整，
但在顧客購買決策上還能更強。

建議改進：

- 強化招牌 / 熱門 / 新品的可見性
- 加入購物車後提供更明顯的回饋
- 讓顧客更容易看見自己已加了哪些品項
- 客製選項畫面降低表單感，提升操作流暢度

---

### G. 批次概念需要更清楚的視覺呈現

系統目前技術上已支援：

- 第 1 批送單
- 第 2 批加點
- 新草稿批次自動建立

但顧客未必知道：

- 後續加點不會改動上一批
- 目前這一頁操作的是哪一批

建議改進：

- 在購物車頁清楚顯示：
  - `目前編輯第 2 批`
  - `第 1 批已送出`
- 在成功頁顯示：
  - `本批已送出`
  - `仍可繼續加點，系統會建立下一批`

---

### H. 畫面信任感仍可提升

點餐系統屬於高頻、低容錯的現場工具。
若畫面太像 demo 或內部工具，
顧客對送單與付款流程的信任感會下降。

建議改進：

- 提高整體品牌一致性
- 加強狀態層次與成功回饋
- 讓關鍵頁面更像正式商用產品，而不是開發測試介面

---

## 3. 建議優先順序

### 第一優先

- 重寫 `合併訂單` 與批次相關顧客文案
- 強化送單成功頁與追單頁資訊
- 明確標示「本批 / 已送出批次 / 可繼續加點」

### 第二優先

- 優化多人共桌的顧客稱呼方式
- 補強異常狀態提示與下一步指引

### 第三優先

- 強化菜單頁購買推力
- 提升整體品牌與信任感

---

## 4. 建議後續落地方式

可將本文件後續拆成三層：

1. 產品層：
   - 哪些改動最影響顧客完成送單與安心感

2. 頁面層：
   - `menu / cart / checkout / success / tracker` 各自要補什麼

3. 工程層：
   - 哪些只是文案調整
   - 哪些需要 API 額外欄位
   - 哪些需要重新設計前端互動

---

## 5. 目前建議的下一步

若要正式進入實作，建議先做：

1. 批次文案調整
2. 送單成功頁與追單頁資訊升級
3. 共桌顧客顯示方式調整

這三項最直接影響顧客是否理解系統、是否安心使用。
