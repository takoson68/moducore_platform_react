# 商品上傳執行書（可執行步驟版）

更新日期：2026-03-06  
範圍：dineCore 商品管理（圖片網址模式，不含本機檔案上傳）

## 0. 執行前確認
1. 確認目前策略為「圖片網址」模式，不做 `multipart/form-data` 上傳。
2. 確認資料欄位使用 `dinecore_menu_items.image_url`。
3. 確認驗收環境可進入商家後台 `menu-admin` 頁面。

完成判定：
- 可在頁面看到商品清單與商品編輯區塊。

## 1. 前端建立商品（含圖片網址）
1. 開啟檔案：`platform/frontend/projects/dineCore/modules/menu-admin/pages/MenuAdminPage.vue`。
2. 確認 `createForm` 內有 `imageUrl` 欄位。
3. 確認新增商品表單有 `type="url"` 輸入欄位，綁定 `createForm.imageUrl`。
4. 確認 `createItem()` 呼叫 `menuAdminStore.createItem()` 時有帶入 `imageUrl`。
5. 確認新增後會重置 `createForm.imageUrl = ''`。

完成判定：
- 可新增商品並帶入圖片網址，新增後清單顯示對應圖片預覽。

## 2. 前端編輯既有商品圖片網址
1. 同檔案確認每筆商品有 `draftImages[item.id]` 編輯欄位。
2. 確認每筆商品有「套用圖片網址」按鈕。
3. 確認按鈕觸發 `saveItemImageUrl(item)`。
4. 確認 `saveItemImageUrl(item)` 呼叫：
   - `menuAdminStore.updateItemImage({ itemId: item.id, imageUrl: draftImages[item.id] })`
5. 確認預覽圖 `img` 綁定使用當前網址值。

完成判定：
- 修改單筆商品圖片網址後，畫面即時反映，且重新載入後仍保留。

## 3. Store 與 Service 串接點
1. 開啟 `platform/frontend/projects/dineCore/modules/menu-admin/store.js`。
2. 確認存在 `updateItemImage` action，且更新目標 item。
3. 開啟 `platform/frontend/projects/dineCore/modules/menu-admin/service.js`。
4. 確認存在 `updateMenuAdminItemImage(payload)`。
5. 目前為雙模式：
   - `apiMode=real`：`load/create-item/update-item-image` 走 `world.http()`。
   - 其餘操作暫走 `mockApiRequest(...)`（待下一輪補齊）。

完成判定：
- 前端按下更新圖片後，store 狀態 `items` 對應 item 的 `imageUrl` 變更成功。

## 4. 錯誤處理最小要求
1. 在 `MenuAdminPage.vue` 的 `runSafely()` 中保留 `window.alert`。
2. 至少覆蓋錯誤：
   - `MENU_ITEM_NOT_FOUND`
   - `MENU_IMAGE_URL_REQUIRED`（若有）
3. 未知錯誤顯示通用訊息並 `console.error(error)`。

完成判定：
- 輸入無效資料時，使用者能收到可理解的錯誤提示。

## 5. 驗收步驟（手動）
1. 進入商家後台商品管理頁。
2. 新增商品，貼上可用圖片網址（例如假圖網站 URL）。
3. 確認商品卡片顯示圖片。
4. 編輯同商品圖片網址為另一張圖並儲存。
5. 重新整理頁面，確認圖片網址變更仍存在。
6. 輸入空網址測試錯誤訊息（依後端或 mock 規則）。

完成判定：
- 新增、更新、重整後資料一致。

## 6. Real API 已執行項目（本次完成）
1. 後端新增：
   - `GET /api/dinecore/staff/menu/items`
   - `POST /api/dinecore/staff/menu/create-item`
   - `POST /api/dinecore/staff/menu/update-item-image`
2. `update-item-image` 會寫入 `dinecore_menu_items.image_url` 並回傳更新後 `item`。
3. 前端 `service.js` 已改為 real/mock 雙模式：
   - real：`world.http()` + `tokenQuery: true`
   - mock：保留既有 `mockApiRequest`
4. 保留前端 URL 模式，不新增 file input。
5. 已完成語法與建置驗證（PHP lint、Vite build）。

完成判定：
- `apiMode=real` 時流程與 mock 模式行為一致。

## 7. 本階段不做
1. 不上傳本機圖片檔。
2. 不建置 `upload-image` 路由。
3. 不實作 `public/uploads/dinecore/menu/` 檔案儲存流程。
