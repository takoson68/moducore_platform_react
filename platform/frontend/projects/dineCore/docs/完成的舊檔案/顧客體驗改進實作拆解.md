# 顧客體驗改進實作拆解

更新日期：2026-03-04

## 1. 文件目的

本文件承接：

- `點餐系統顧客體驗改進清單.md`

將其中偏產品 / 體驗層的建議，
往下拆成可實際排程與落地的工程項目。

本文件重點不是重新討論問題，
而是回答：

- 先做什麼
- 改哪些頁
- 哪些只要文案
- 哪些需要前端互動調整
- 哪些可能需要 API 補欄位

---

## 2. 建議執行順序

### Phase 1：低風險高回報

目標：

- 先用最少改動，提升顧客理解與安心感

內容：

- 文案調整
- 成功頁資訊補強
- 購物車 / 確認頁批次提示

### Phase 2：中等改動

目標：

- 改善共桌與批次的顯示邏輯

內容：

- 共桌顧客顯示方式
- 批次摘要卡
- 追單頁結構優化

### Phase 3：體驗升級

目標：

- 讓整體更像正式商用點餐產品

內容：

- 菜單頁轉化優化
- 品牌感 / 信任感提升
- 更精緻的操作回饋

---

## 3. 任務拆解

### Task A1：重寫 `合併訂單` 文案

優先級：高
狀態：已完成

影響頁面：

- `cart`

建議方向：

- 將 `合併訂單` 改成更直覺的顧客語言
- 建議首選：`確認本批餐點`

完成標準：

- 顧客不需要理解系統模型，也知道按下去是前往送單確認

---

### Task A2：補強購物車頁的批次提示

優先級：高
狀態：已完成

影響頁面：

- `cart`

建議方向：

- 明確顯示：
  - 目前正在編輯第幾批
  - 前一批是否已送出
  - 這一批送出後不會回頭改動前一批

完成標準：

- 顧客在購物車頁就能理解目前內容屬於哪一批

是否需要 API：

- 可能不需要新增 API
- 先確認現有 `batches` / `current batch` 欄位是否已足夠

---

### Task A3：補強確認頁的送單說明

優先級：高
狀態：已完成

影響頁面：

- `checkout`

建議方向：

- 在送單前再次提醒：
  - 目前送出的是第幾批
  - 送出後這一批會鎖定
  - 仍可於送出後繼續加點下一批

完成標準：

- 顧客在送單前對後續結果有明確認知

---

### Task A4：升級送單成功頁

優先級：高
狀態：已完成

影響頁面：

- `checkout success`

建議方向：

- 明確顯示：
  - 第幾批已送出
  - 本批品項數
  - 本批金額
  - 店家已收到
  - 預估等待時間

完成標準：

- 顧客送單完成後能立即得到安心感與下一步理解

是否需要 API：

- 可能需要確認 success payload 是否已包含足夠資訊
- 若不足，再補最小必要欄位

---

### Task A5：升級追單頁

優先級：高
狀態：已完成

影響頁面：

- `order-tracker`

建議方向：

- 更清楚顯示：
  - 各批次順序
  - 各批次狀態
  - 最近送出的批次
  - 是否仍可繼續加點

完成標準：

- 顧客能直觀看懂不是只有一張平面訂單，而是多批次進度

---

### Task B1：重新設計多人共桌顯示名稱

優先級：中

影響頁面：

- `cart`
- `checkout`
- `order-tracker`

建議方向：

- 弱化 `1號顧客 / 2號顧客`
- 第一版可先改成：
  - 目前使用者顯示為 `你的餐點`
  - 其他顧客顯示為 `同桌顧客`

進一步版本：

- 允許快速自訂暱稱

完成標準：

- 共桌資訊仍能辨識，但不再太像內部技術標記

---

### Task B2：補強異常狀態頁與錯誤提示

優先級：中

影響頁面：

- `entry unavailable`
- `menu`
- `cart`
- `checkout`

建議方向：

- 所有顧客會看到的異常訊息統一成：
  - 簡短
  - 可理解
  - 帶下一步建議

完成標準：

- 顧客遇到失敗時不會只感覺「系統壞了」

---

### Task C1：強化菜單頁購買推力

優先級：中

影響頁面：

- `menu`

建議方向：

- 強化招牌 / 熱門 / 新品標示
- 提高加入購物車後的回饋
- 讓已加入的品項更容易被看見
- 降低客製選項的表單感

完成標準：

- 顧客在菜單頁做決定更快
- 不容易遺漏自己已經加了什麼

---

### Task C2：提升整體品牌與信任感

優先級：中

影響頁面：

- `entry`
- `menu`
- `cart`
- `checkout`
- `success`
- `order-tracker`

建議方向：

- 統一視覺語氣
- 提升成功 / 狀態 / 等待的視覺層次
- 降低 demo 感

完成標準：

- 顧客從第一眼就覺得這是正式可用的商用點餐系統

---

## 4. 頁面對照表

| 頁面 | 建議先做 |
|---|---|
| `menu` | 熱門 / 新品 / 加入回饋 / 已選品項可見性 |
| `cart` | 批次提示、文案重寫、共桌顯示優化 |
| `checkout` | 送單前說明、本批資訊清楚化 |
| `checkout success` | 成功後安心資訊升級 |
| `order-tracker` | 多批次進度顯示強化 |
| `entry unavailable` | 異常提示更明確 |

---

## 5. 哪些任務可直接開始

### 不需要動 API，可先做

- `合併訂單` 文案調整
- 購物車 / 確認頁批次說明
- `EntryUnavailablePage` 與顧客錯誤文案優化
- 共桌顧客顯示文案第一版調整

### 可能需要確認現有 payload 是否足夠

- 成功頁資訊升級
- 追單頁批次摘要強化
- 批次層級提示與等待時間呈現

### 偏設計 / UI 優化

- 菜單頁購買推力
- 品牌信任感提升

---

## 6. 建議直接開始的第一輪實作

若要從這份文件直接進入開發，
建議第一輪做以下三項：

1. `cart`：把 `合併訂單` 改為 `確認本批餐點`，並補批次提示
2. `checkout success`：補本批送單成功資訊
3. `order-tracker`：補批次結構與已送出批次理解

這三項最接近顧客感知，也最能快速拉高體驗品質。
