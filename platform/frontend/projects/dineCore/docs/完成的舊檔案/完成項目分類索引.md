# 完成項目分類索引

更新日期：2026-03-04

## 1. 文件目的

本文件用於整理 DineCore `docs/` 中：

- 已完成並仍具參考價值的文件
- 已完成且已歸檔到 `完成的舊檔案/` 的文件

重點不是列出所有檔名，
而是提供「完成項目目前應如何被理解」的分類入口。

## 5. 建議閱讀方式

### 若要理解目前系統現況

優先讀：

1. `README.md`
2. `DINECORE_CURRENT_STATE.md`
3. `DINECORE_PROGRESS_TRACKER.md`

### 若要理解目前有效規格

依任務讀對應契約或邊界文件，例如：

- session / 顧客身份：讀 guest ordering session 文件
- reports：讀 reports 契約與 boundary
- audit-close：讀 audit-close 契約與 boundary

### 若要追溯某個已完成的 batch 重構歷程

再進入：

- `完成的舊檔案/`

按以下順序回看：

1. 資料表調整草案
2. API 調整草案
3. 前端頁面切換規則
4. 任務拆解 / 執行書

---

## 6. 整理原則

未來若再整理 `docs/`，建議維持以下規則：

- 仍會影響目前工程判斷的文件，留在 `docs/` 根目錄
- 階段性規劃、拆解、草案、執行書，移到 `完成的舊檔案/`
- 歸檔後仍應有一份分類索引，避免只剩資料夾堆積
