# 狀態模型定義

## 文件定位
本文件用於定義餐廳地圖產生器模組的 editor state，避免實作過程中狀態責任混亂。

## 1. 狀態分層原則

- 正式資料與編輯中暫存狀態必須分離。
- 地圖資料、桌位資料、模式狀態、工具狀態不可混成單一無邊界結構。
- `draftState` 僅代表編輯中狀態，不等同正式資料。

## 2. 主資料狀態

### 2.1 Map 狀態

- `maps`
- `activeMapId`

### 2.2 Map 內容狀態

- `map`
- `objects`
- `tables`

補充原則：

- 第一階段可先以 `map + objects` 為主。
- 第二階段再擴充 `tables`。

## 3. 編輯器狀態

### 3.1 模式狀態

- `viewMode`
- `editMode`
- `mapEditMode`
- `tableEditMode`
- `objectEditMode`

### 3.2 工具狀態

- `activeTool`
- `toolbarLocked`

### 3.3 選取狀態

- `activeObjectId`
- `activeTableId`

### 3.4 編輯中暫存狀態

- `pendingPolyline`
- `pendingShape`
- `pendingText`
- `hoveredSegmentIndex`
- `draftDirty`

## 4. 狀態責任

- `activeMapId` 決定目前查看與編輯的地圖。
- `activeObjectId` 僅在 `Map Edit Mode` 下使用。
- `activeTableId` 僅在 `Table Edit Mode` 下使用。
- `pendingPolyline` 僅在折線建立中存在。
- `hoveredSegmentIndex` 僅在 `Polyline` 的 `Object Edit Mode` 下使用。
- `toolbarLocked` 僅在 `Object Edit Mode` 或繪製中狀態下啟用。

## 5. 狀態切換原則

- 切換 `active map` 前，需先處理未儲存變更。
- 切換 `Map Edit Mode` / `Table Edit Mode` 時，需清除前一模式 active 狀態。
- 離開 `Object Edit Mode` 時，需同步解除 `toolbarLocked`。
- 刪除 `active map` 時，需清除該 map 相關的編輯中暫存狀態。

## 6. 狀態落地原則

- 正式儲存只保存已完成資料。
- 草稿存檔可保存 `draftState`。
- 內部狀態格式可與交換格式不同，但需由轉換層保持語意一致。
