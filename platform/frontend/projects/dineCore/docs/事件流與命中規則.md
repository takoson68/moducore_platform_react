# 事件流與命中規則

## 文件定位
本文件用於定義餐廳地圖產生器模組中的事件處理順序、命中規則與模式邊界。

## 1. 命中分層原則

- `Map Layer` 只處理地圖圖形事件。
- `Table Layer` 只處理桌位事件。
- `UI Layer / Overlay Layer` 只處理 UI 控制事件。
- 命中判定需先看當前模式，再決定可操作集合。

## 2. 模式與事件對應

### 2.1 View Mode

- 僅查看。
- 點擊物件不進入編輯。

### 2.2 Map Edit Mode

- 命中 `objects`。
- 點擊空白區時，若有工具啟用則建立新物件；否則不動作。
- 點擊地圖物件可進入 `Object Edit Mode`。

### 2.3 Table Edit Mode

- 命中 `tables`。
- 點擊空白區時，若有 `Table` 工具啟用則建立桌位；否則不動作。
- 點擊桌位可進入 `Object Edit Mode`。

## 3. 空白區規則

- 空白區不自動退出 `Object Edit Mode`。
- 空白區是否建立新物件，取決於當前模式與是否已有工具啟用。
- 點空白區不應跨模式清理不相關狀態。

## 4. Object Edit Mode 命中優先序

- 控制點命中優先於物件本體。
- 物件本體命中優先於空白區。
- 若圖層重疊，仍需先遵守模式邊界，再看 `render order`。
- 在 `Polyline` 的 `Object Edit Mode` 中，segment `hover` 命中僅作為曲線控制點顯示依據，不改變 `activeObjectId`。
- 在 `Polyline` 的 `Object Edit Mode` 中，同一時間只允許一個 `hoveredSegmentIndex` 生效。
- 僅目前被 `hover` 的 segment 可顯示空心控制點。
- 直線段被 `hover` 時，空心控制點顯示於線段中點。
- 曲線段被 `hover` 時，空心控制點顯示於該段控制點位置。
- 拖曳空心控制點後，應將事件轉換為該段曲線控制點編輯，而非一般新增節點行為。

## 5. 模式切換清理規則

- 切換 `Map Edit Mode` / `Table Edit Mode` 前，需先退出當前 `Object Edit Mode`。
- 模式切換時，需清除 `activeObjectId` 或 `activeTableId`。
- 模式切換後，命中邏輯立即切到對應集合。

## 6. 刪除與切換事件

- 刪除物件後，需退出對應 `Object Edit Mode`。
- 刪除 `active map` 後，需清除相關 `draftState` 與 working mode。
- 切換地圖、離開頁面、刪除地圖前，若有未儲存變更需先提示。

## 7. SVG 命中原則

- 優先以 `SVG` 元素與資料標記進行命中。
- 不自行模擬 HTML `<canvas>` 式命中系統。
- 所有事件最終都需轉回世界座標處理。
