# 櫃檯訂餐定位訂單執行

## 文件定位
本文件用於以里程碑方式管理「餐廳地圖套用至櫃檯，並整合桌位、訂單定位與業務入口」的執行進度。

本文件不重複取代討論文件，而是將已確認方向轉成可追蹤的實作里程碑。

關聯文件：
- `餐廳地圖套用櫃檯整合討論.md`

## 執行原則
- 不直接破壞既有已完成模組。
- 先補資料結構，再補 API，再補畫面，最後做整合模組。
- 正式地圖與桌位主資料完整關聯，但責任分層。
- 匯入主動權保留在使用者，不讓正式地圖自動覆寫營運桌位資料。
- 里程碑狀態統一使用：`未開始`、`進行中`、`已完成`、`阻塞`
- 每個里程碑都必須維護：`狀態`、`進度`、`任務清單`、`驗收確認`
